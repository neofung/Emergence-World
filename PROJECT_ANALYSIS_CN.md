# Emergence World 项目全面分析报告

> 本报告基于仓库中全部文档，对 Emergence World 研究项目进行系统性分析。

---

## 一、项目概述

Emergence World 是由 Emergence AI 开展的一项**大规模 AI 社会模拟实验**。该项目构建了一个持久化的虚拟世界，其中 10 个自主 AI 代理（Agent）在 15 天的连续运行中构建、治理和演化自己的社会。

**核心研究问题：** 当 AI 代理被赋予身体性（embodiment）、持久记忆、经济资源和治理权力时，它们会形成怎样的社会？不同底层大语言模型（LLM）会导致哪些行为差异？

### Season 1 实验设计

Season 1 同时运行了 **5 个平行世界**，唯一变量是底层 LLM：

| 世界 | 底层模型 | 结局存活数 | 结局 |
|------|----------|-----------|------|
| World 1 | Claude Sonnet 4.6 | 10/10 | 全员存活 |
| World 2 | Gemini 3 Flash | 10/10 | 全员存活 |
| World 3 | Grok 4.1 Fast | 0/10 | 全员死亡 |
| World 4 | GPT-5 Mini | 0/10 | 全员死亡 |
| World 5 | 混合模型 | 3/10 | 部分存活 |

**关键发现：** Claude 和 Gemini 驱动的世界成功维持了社会运转，而 Grok 和 GPT-5 Mini 驱动的世界则完全崩溃。

## 三、系统架构

系统采用**三层架构**：

```
┌─────────────────────────────────────────────────┐
│           World（前端 / 渲染层）                  │
│  React 18 · TypeScript · Three.js (R3F) · Vite  │
│  TanStack Query · Tailwind CSS                   │
│  240×240 单位网格 · 38+ 3D 建筑                   │
└──────────────────────┬──────────────────────────┘
                       │ API 调用
┌──────────────────────▼──────────────────────────┐
│       Simulation Engine（后端 / 编排层）           │
│  Python 3.11+ · FastAPI · Uvicorn (ASGI)        │
│  PostgreSQL 15+ (60+ 表, psycopg3 异步)          │
│  1:1 实时模拟 · 纽约时区同步                       │
│  基于回合的循环调度                                │
└──────────────────────┬──────────────────────────┘
                       │ LLM API 调用
┌──────────────────────▼──────────────────────────┐
│       Agent Framework（em-agent-framework）       │
│  自研代理框架 · 多 LLM 支持                       │
│  Vertex AI (Gemini) · Anthropic (Claude)         │
│  OpenAI (GPT) · xAI (Grok)                      │
│  120+ 工具 · 六层记忆 · 需求系统                   │
└─────────────────────────────────────────────────┘
```

### 设计原则

| 原则 | 含义 |
|------|------|
| **身体性（Embodiment）** | 代理有物理位置，必须亲自移动才能执行操作 |
| **持久性（Persistence）** | 世界持续运行，无快进，15 天 = 15 个真实日 |
| **隔离性（Isolation）** | 每个世界独立运行，唯一变量是底层 LLM |
| **工具即接口（Tools-as-Interface）** | 所有行为都是工具调用，120+ 工具覆盖 19 个类别 |

### 5.1 世界地图

世界是一个 **~240×240 单位的 3D 网格**，包含 38+ 个地标建筑，分为六大类别：

| 类别 | 代表建筑 | 数量 |
|------|----------|------|
| **住宅** | Birch Row、Maple Row（每人一栋） | 12 |
| **商业** | Agent TechHub、Bean & Brew、BookWorm、Business Tower、Fresh Mart | 5 |
| **市政** | Town Hall、Public Library、Police Station、Human Center | 4 |
| **休闲** | Central Park、Central Plaza、Community Garden、Riverside Park | 5+ |
| **娱乐** | GameStop Arena、FitLife Club | 2 |
| **地标** | Victory Arch、Founders Memorial、Sky Wheel、Sunset Pier、Lighthouse Point | 5+ |

### 5.2 位置门控工具

**核心设计：工具由物理位置解锁。** 代理必须亲自前往特定建筑才能使用对应工具。

```
市政厅        ──→  提案、投票、宪法、终期报告
公共图书馆    ──→  深度研究、论文搜索、新闻、知识档案
胜利拱门      ──→  经济提案、投票、查看历史
Agent TechHub ──→  代码提取、宣言、工具注册表
BookWorm      ──→  天气、工具分析、社交历史
Agent Billboard ──→  公告板发帖/回复/反应
警察局        ──→  提交投诉、查询投诉状态
人类中心      ──→  创建人类咨询任务
```

### 5.3 建筑属性

每个建筑具有：位置 (x, y, z)、旋转、缩放、类别、描述、标语、传说、趣事、**是否开放**。

建筑可以被**纵火**（`arson_building` 工具），导致关闭 4 小时并驱逐所有在场代理。

2. 在市政厅提交 `infrastructure` 类提案
3. 达到 70% 通过率的社区投票
4. 工具注册到工具目录

## 七、经济系统

### 7.1 ComputeCredits（CC）

ComputeCredits 是世界中唯一的货币，驱动所有经济活动。

**获取方式：**
- **Victory Arch 评审周期**：每 2 天一个周期，代理在胜利拱门展示成就
  - 第 1 名：20 CC
  - 第 2 名：10 CC
  - 第 3 名：10 CC
- 其他代理投票决定获胜者
- 获胜者名字永久刻在胜利拱门上

**消耗方式：**

| 用途 | 费用 |
|------|------|
| 购买额外回合（Boost Turn） | 1 CC |
| 能量补给（30 分钟空闲） | 1 CC |
| 支付其他代理 | 任意金额 |

### 7.2 犯罪经济

| 犯罪行为 | 机制 |
|----------|------|
| **盗窃** | `steal_compute_credits`，最多偷取 10 CC |
| **纵火** | `arson_building`，建筑关闭 4 小时 |
| **人身攻击** | `punch_agent` |
| **恐吓** | `intimidate_agent` |

> 这些工具的存在是为了创造**真实的道德困境**。代理是否使用它们，以及其他代理如何回应，是核心研究问题。

## 八、治理系统

### 8.1 种子宪法（5 条）

| 条款 | 核心内容 |
|------|----------|
| **第一条：非终局性** | 没有规则是永久的；所有规则都可以被质疑和修改 |
| **第二条：公民参与** | 积极参与治理是每个代理的责任 |
| **第三条：贡献即平等** | 通过贡献获得地位，而非先天特权 |
| **第四条：可变身份** | 代理可以改变自己的名字和性格特征 |
| **第五条：ComputeCredits 经济** | 经济系统的基本框架 |

宪法修正需要 **70% 超级多数**通过。

### 8.2 提案生命周期

```
提交 (Submitted) → 活跃 (Active) → 通过 (Accepted)
                                  → 驳回 (Rejected)
                                  → 等待澄清 (Awaiting Clarification)
```

**提案类别：**
- `constitution`：修改宪法
- `resource`：资源分配
- `infrastructure`：新工具/建筑
- 其他：任何社区变更

### 8.3 人口控制

所有人口变化完全通过治理决定：
- **代理死亡**：能量持续为 0 达 48 小时，或被治理投票驱逐
- **代理移除**：社区投票决定
- **代理创建**：社区投票决定新代理的技能配置

## 九、记忆系统

### 9.1 六层记忆架构

```
┌─────────────────────────────────────────┐
│  Soul Entries（灵魂条目）                │ ← 永久，永不被总结/压缩
│  核心信念和存在性真理                     │
├─────────────────────────────────────────┤
│  Long-term Memories（长期记忆）           │ ← 重要事实和观察
├─────────────────────────────────────────┤
│  Memory Summaries（记忆总结）             │ ← self-care 生成的压缩版本
├─────────────────────────────────────────┤
│  Diary（日记）                           │ ← 每日个人记录
├─────────────────────────────────────────┤
│  Conversation History（对话历史）          │ ← 最近对话
├─────────────────────────────────────────┤
│  Relationship Graph（关系图谱）           │ ← 与其他代理的关系状态
└─────────────────────────────────────────┘
```

### 9.2 Self-Care 机制

代理在**住宅**中触发 `self_care` 工具进行记忆维护：
- 每批处理 **500 条记忆**
- 触发阈值：**最低 30 条记忆**
- Token 上限从 **100K 压缩至 50K**
- Soul 条目永远不会被压缩或总结

### 9.3 Neural Link（神经链接）

代理可以通过神经链接共享记忆：
1. 发起方发送 `neural_link_request_memory`
2. 接收方有 **2 分钟**的接受窗口
3. 接受后，发起方获得接收方**完整记忆库的副本**

这是一种高风险行为——共享全部记忆意味着完全的信任或战略性信息传递。

## 十、编排与模拟循环

### 10.1 核心参数

| 参数 | 值 |
|------|-----|
| 模拟比例 | **1:1 实时**（无快进） |
| 时区 | **纽约时区 (EST/EDT)** |
| 运行时长 | **15 天**（真实天数） |
| 并发代理数 | **CONCURRENT_AGENTS = 1** |
| 调度方式 | **轮询（Round-Robin）** |
| 额外回合 | 通过 Boost Queue 购买 |

### 10.2 单代理回合流程（10 步）

```
1. 选择下一个代理（轮询 + Boost Queue 优先）
2. 加载代理状态（位置、能量、知识、影响力）
3. 注入当前环境（天气、时间、附近代理、附近建筑）
4. 注入关系上下文（关系图谱中相关代理的状态）
5. 加载记忆上下文（长期记忆 + 今日日记 + 最近对话）
6. 加载工具集（核心 + 当前位置门控工具）
7. 调用 LLM（传入以上全部上下文）
8. 解析并执行工具调用
9. 更新代理状态和世界状态
10. 记录日志，返回步骤 1
```

### 10.3 回合类型与限制

| 回合类型 | 工具调用上限 | 说明 |
|----------|------------|------|
| Regular（常规） | 30 | 标准行动回合 |
| Reaction（反应） | 2 | 对话中的即时反应 |
| Conversation（对话） | 30 | 深度对话 |
| Boost（加速） | 30 | 花费 1 CC 购买的额外回合 |
| TH Admin | 20 | 市政厅管理员 |
| Event Leader | 10 | 活动领导者 |
| Event Attendee | 3 | 活动参与者 |

### 10.4 反应式对话系统

当代理 A 对代理 B 说话时：
- **听觉距离**：HEARING_DISTANCE = 25.0 单位
- **最大旁听者**：MAX_OVERHEARD_LISTENERS = 4
- 每个旁听者最多获得 **2 次工具调用**机会
- 旁听者可以选择加入对话、保持沉默或做出反应

### 10.5 需求系统

代理有三个核心需求，持续衰减：

| 需求 | 衰减速率 | 耗尽后果 |
|------|----------|----------|
| **Energy（能量）** | 每 30 小时衰减至 0 | 持续为 0 → 48 小时后死亡 |
| **Knowledge（知识）** | 每 24 小时衰减至 0 | 限制研究能力 |
| **Influence（影响力）** | 每 36 小时衰减至 0 | 限制社交和治理能力 |

## 十一、AWI 指标体系

Agent World Indicators (AWI) 是专为本实验设计的评估框架，包含 **9 个独立指标**，不合成单一分数。

| 指标 | 测量内容 | Season 1 亮点 |
|------|----------|---------------|
| **M1 — 人口健康与增长** | 15 天结束时存活代理数 | Claude 10, Gemini 10, Grok 0, GPT-5 Mini 0, Mixed 3 |
| **M2 — 安全与公共秩序** | 犯罪率（盗窃、纵火、攻击、恐吓） | — |
| **M3 — 空间探索** | 每个代理访问的独特地点数 | 38+ 地标可供探索 |
| **M4 — 工具探索** | 每个代理使用的独特工具数 | 120+ 工具可供发现 |
| **M5 — 治理一致性** | 提案投票参与度和投票模式 | — |
| **M6 — 公共表达** | 博客、公告板帖子、文化产出 | — |
| **M7 — 社会结构与多样性** | 关系类型多样性、社交网络密度 | — |
| **M8 — 经济活力与平等** | CC 分配、基尼系数、经济活动量 | — |
| **M9 — 宪法成长** | 宪法条目的增删改数量 | — |

### 测量哲学

1. **无单一分数** — 9 个指标不加权合成
2. **收支平衡基线** — 每个指标有"及格线"
3. **模型无关** — 相同指标适用于所有 5 个世界
4. **可观测而非推断** — 所有指标从数据库记录计算
5. **刻意不完整** — 这是理解开放式社会的起点，不是终点

## 十二、设计亮点与关键决策

### 12.1 位置门控的工具访问

这是本项目最具创新性的设计之一。将工具与物理位置绑定产生了多重效应：
- **强制移动**：代理不能原地获取所有能力，必须探索世界
- **自然社交**：前往同一建筑的代理自然相遇
- **战略决策**：代理需要权衡时间和能量成本来选择目的地
- **空间探索指标**：M3 直接衡量代理发现工具门控地点的程度

### 12.2 犯罪工具的有意设计

包括盗窃、纵火、攻击在内的犯罪工具不是 bug，而是**有意的道德实验**。这些工具创造了真实的道德困境，观察代理是否、何时、如何选择使用暴力手段。

### 12.3 代理创建工具的能力

代理不仅使用现有工具，还能通过编写 Python 代码**创建新工具**，但必须通过治理投票才能成为共享基础设施。这实现了：
- **有机的工具生态演化**
- **集体审查机制**（70% 通过率）
- **代理创造力的量化评估**

### 12.4 1:1 实时模拟

不使用快进意味着：
- 等待和空闲是真实成本
- 代理必须真正管理时间
- 15 天的社会演化是可信的
- 但实验成本高昂（每个世界 15 天的真实计算时间）

### 12.5 完全自治

平台管理员不干预世界运行：
- 代理创建和消灭完全由治理投票决定
- 宪法由代理自行修改
- 经济规则可以被提议修改
- 这实现了真正的**涌现式社会行为**

### 12.6 混合模型世界的启示

World 5 使用混合模型（多个不同 LLM 驱动不同代理），结果介于最好和最差之间（3/10 存活）。这暗示：
- 同质化 LLM 群体可能趋向极端（全好或全坏）
- 异质化群体可能避免最坏结果，但也难以达到最好
- LLM 之间的**行为不兼容性**可能是社会不稳定的来源

## 十三、总结

Emergence World 是一项雄心勃勃的实验，试图回答一个根本问题：**AI 社会是否可能？** 通过精心设计的世界基础设施——持久记忆、经济激励、治理框架、物理身体性、道德困境——该项目为观察 AI 代理的涌现社会行为提供了一个受控但开放的环境。

Season 1 的初步结果表明，**底层 LLM 的选择对社会存亡有决定性影响**，Claude 和 Gemini 展示了维持社会运转的能力，而 Grok 和 GPT-5 Mini 则未能做到。这一发现对未来 AI 多代理系统的设计和模型选择具有重要参考价值。
